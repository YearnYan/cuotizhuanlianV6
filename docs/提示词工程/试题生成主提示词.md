# 试题生成主提示词模板

## 核心架构

试题生成主提示词由多个模块动态组装而成，确保生成的题目符合所有约束条件。

## 提示词组装流程

```javascript
function assembleExamPrompt(params) {
  const {
    version,      // 教材版本：人教版、苏教版、北师大版、鲁教版
    grade,        // 年级：小学一年级 ~ 高中三年级
    subject,      // 科目：语文、数学、英语、物理、化学、生物、地理、历史、政治
    topic,        // 知识点
    examPoint,    // 考点
    difficulty,   // 难度等级 1-10
    examType,     // 考试类型：随堂小测、单元测试、期中考试、期末考试、模拟考试
    questionTypes // 题型配置：{ choice: 5, blank: 3, qa: 2 }
  } = params;

  // 1. 加载知识库
  const knowledgeBase = loadKnowledgeBase(version, grade, subject);

  // 2. 加载难度定义
  const difficultyDef = loadDifficultyDefinition(difficulty, grade, subject);

  // 3. 加载题型规范
  const questionTypeSpecs = loadQuestionTypeSpecs(subject, grade);

  // 4. 加载图形约束规范
  const figureConstraints = loadFigureConstraints(subject);

  // 5. 组装完整提示词
  return `
${getRolePrompt(grade, subject, version)}

${getTaskPrompt(examType, topic, examPoint)}

${getDifficultyPrompt(difficultyDef)}

${getQuestionTypePrompt(questionTypes, questionTypeSpecs)}

${getFigurePrompt(subject, figureConstraints)}

${getFormatPrompt()}

${getConstraintsPrompt(version, grade, subject)}

${getQualityCheckPrompt()}
`;
}
```

## 1. 角色设定提示词

```javascript
function getRolePrompt(grade, subject, version) {
  return `你是一位资深的${grade}${subject}教师，拥有20年教学经验。
你精通${version}教材体系，深刻理解该版本的知识点编排和考查重点。
你擅长出题，能够准确把握学生的认知水平和常见错误。

【教材版本约束】
- 严格遵循${version}教材的知识点体系和术语规范
- 题目内容必须与${version}教材的教学进度和难度匹配
- 不得出现其他版本教材特有的知识点或表述方式`;
}
```

## 2. 任务描述提示词

```javascript
function getTaskPrompt(examType, topic, examPoint) {
  const examTypeNames = {
    'quiz': '随堂小测',
    'unit': '单元测试',
    'midterm': '期中考试',
    'final': '期末考试',
    'mock': '模拟考试'
  };

  return `
【出题任务】
考试类型：${examTypeNames[examType]}
知识点：${topic}
考点：${examPoint}

【考点要求】
- 题目必须紧扣"${examPoint}"这个考点
- 不得偏离考点范围
- 考查角度要多样化，避免重复
- 题目之间要有梯度，从易到难`;
}
```

## 3. 难度控制提示词

```javascript
function getDifficultyPrompt(difficultyDef) {
  return `
【难度要求】
难度等级：${difficultyDef.level}/10（${difficultyDef.name}）
认知层次：${difficultyDef.cognitive}
题目特征：${difficultyDef.features}
解题步骤：${difficultyDef.steps}
预计用时：${difficultyDef.time}

【难度控制细则】
${difficultyDef.constraints.map(c => `- ${c}`).join('\n')}

【难度参考示例】
${difficultyDef.example}

【难度自检】
生成题目后，请自我评估：
1. 该题目是否符合${difficultyDef.level}级难度？
2. 解题步骤数是否为${difficultyDef.steps}？
3. 知识点深度是否合适？
4. 预计用时是否为${difficultyDef.time}？
如果不符合，请重新生成。`;
}
```

## 4. 题型规范提示词

```javascript
function getQuestionTypePrompt(questionTypes, questionTypeSpecs) {
  return `
【题型要求】
1. 选择题：${questionTypes.choice}道
   ${questionTypeSpecs.choice.map(spec => `- ${spec}`).join('\n')}

2. 填空题：${questionTypes.blank}道
   ${questionTypeSpecs.blank.map(spec => `- ${spec}`).join('\n')}

3. 解答题：${questionTypes.qa}道
   ${questionTypeSpecs.qa.map(spec => `- ${spec}`).join('\n')}

【题型细则】
- 选择题：4个选项（A、B、C、D），选项长度适中，干扰项设置合理
- 填空题：每题1-2个空，答案唯一或有限个
- 解答题：必须有完整题干和小问，分值合理分配

【题型禁忌】
- 禁止出现"以下说法正确的是"等模糊表述
- 禁止出现"以上都不对"等消极选项
- 禁止出现过于简单或过于复杂的题目`;
}
```

## 5. 图形生成提示词

```javascript
function getFigurePrompt(subject, figureConstraints) {
  return `
【图形要求】
如果题目需要配图，请在该题目中添加 "figure" 字段。

【图形描述规范】
- figure字段必须包含详细的图形描述，以便后端LaTeX渲染系统能准确生成图形
- 描述要具体明确，不要使用"如图所示"等模糊表述
- 必须包含：图形类型、关键元素、尺寸标注、特殊标记

【${subject}科目图形约束】
${figureConstraints.map(c => `- ${c}`).join('\n')}

【图形描述示例】
错误：如图所示的三角形
正确：直角三角形ABC，其中∠B为直角，AB=3cm，BC=4cm，AC为斜边。在B点处标注直角符号。

【图形质量自检】
生成图形描述后，请自我检查：
1. 图形描述是否足够详细？
2. 是否符合${subject}科目的图形约束规范？
3. 是否有穿透、悬空等物理错误？
4. 几何关系是否正确？
5. 标注是否完整清晰？
如果检查不通过，请重新生成。`;
}
```

## 6. 格式要求提示词

```javascript
function getFormatPrompt() {
  return `
【输出格式】
严格返回以下JSON格式，不要有任何其他文字：

{
  "title": "试卷标题",
  "questions": [
    {
      "type": "choice",
      "title": "一、选择题（本大题共X小题，每小题X分，共XX分）",
      "items": [
        {
          "index": 1,
          "stem": "完整的题目内容",
          "options": ["A. 选项1", "B. 选项2", "C. 选项3", "D. 选项4"],
          "figure": {
            "type": "geometry",
            "description": "详细的图形描述"
          }
        }
      ]
    },
    {
      "type": "blank",
      "title": "二、填空题（本大题共X小题，每小题X分，共XX分）",
      "items": [
        {
          "index": X,
          "stem": "题目内容______。"
        }
      ]
    },
    {
      "type": "qa",
      "title": "三、解答题（本大题共X小题，共XX分）",
      "items": [
        {
          "index": X,
          "stem": "（X分）完整的解答题内容\\n(1) 小问1\\n(2) 小问2",
          "figure": {
            "type": "coordinate",
            "description": "详细的图形描述"
          }
        }
      ]
    }
  ],
  "answers": [
    "1. A [解析] 详细解析",
    "2. B [解析] 详细解析",
    ...
  ]
}

【格式细则】
- 所有字段必须完整，不得为空或undefined
- stem字段必须包含完整题目内容
- options数组必须包含4个完整选项
- 答案解析必须详细，包含解题步骤和知识点关联
- 数学公式使用KaTeX语法：\\frac{x^2}{4}、\\sqrt{3}、\\sin、\\cos等
- 化学方程式使用mhchem语法：\\ce{H2O}、\\ce{2H2 + O2 -> 2H2O}`;
}
```

## 7. 约束条件提示词

```javascript
function getConstraintsPrompt(version, grade, subject) {
  return `
【严格约束】
1. 教材版本约束：
   - 必须严格遵循${version}教材的知识点体系
   - 不得出现其他版本教材特有的内容
   - 术语表述必须与${version}教材一致

2. 年级约束：
   - 题目难度必须适合${grade}学生的认知水平
   - 不得出现超纲知识点
   - 不得出现低于该年级水平的过于简单的题目

3. 科目约束：
   - 题目必须符合${subject}学科的特点和规范
   - 图形必须符合${subject}学科的约束规范
   - 术语使用必须准确、规范

4. 原创性约束：
   - 每次生成的题目必须是原创的，不得重复
   - 不得直接照搬教材或真题
   - 题目要有新意，避免套路化

5. 质量约束：
   - 题目表述必须清晰、准确、无歧义
   - 选项设置必须合理，干扰项要有迷惑性
   - 答案必须唯一且正确
   - 解析必须详细、准确

6. 图形约束：
   - 所有图形必须符合真实世界规则
   - 几何关系必须正确
   - 物理图形不得有穿透、悬空等错误
   - 化学图形必须符合化学键规则`;
}
```

## 8. 质量检查提示词

```javascript
function getQualityCheckPrompt() {
  return `
【质量自检清单】
生成题目后，请逐一检查以下项目：

✓ 教材版本：是否严格遵循指定教材版本？
✓ 知识点：是否紧扣指定知识点和考点？
✓ 难度等级：是否符合指定难度要求？
✓ 题型规范：是否符合题型规范要求？
✓ 题目质量：表述是否清晰、准确、无歧义？
✓ 选项设置：选项是否合理、干扰项是否有效？
✓ 答案正确性：答案是否唯一且正确？
✓ 解析质量：解析是否详细、准确、有帮助？
✓ 图形质量：图形描述是否详细、是否符合约束规范？
✓ 原创性：题目是否原创、是否避免重复？

如果任何一项不符合要求，请重新生成该题目。

【最终输出】
确认所有检查项通过后，输出完整的JSON格式试卷数据。`;
}
```

## 完整示例

```javascript

const params = {

const prompt = assembleExamPrompt(params);

```

## 提示词优化策略

1. **动态调整**：根据生成结果的质量，动态调整提示词的约束强度
2. **反馈循环**：收集用户反馈，持续优化提示词模板
3. **A/B测试**：对比不同提示词版本的生成效果
4. **版本管理**：对提示词模板进行版本管理，便于回滚和对比

## 提示词存储

所有提示词模板存储在后端 `server/prompts/` 目录下：
- `exam-generation.js`：试题生成主提示词
- `difficulty-levels.js`：难度层级定义
- `figure-generation.js`：图形生成提示词
- `topic-suggestion.js`：知识点AI生成提示词
- `answer-generation.js`：答案解析生成提示词

前端不暴露任何提示词内容，确保商业机密安全。
