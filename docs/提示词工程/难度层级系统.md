# 难度层级系统设计

## 总体原则

1. **10级难度必须有明显区分度**：用户能清晰感受到1级和10级的巨大差异
2. **年级×科目矩阵细化**：同样是"5级难度"，小学数学和高中数学的实际难度完全不同
3. **认知层次对应**：基于布鲁姆认知分类法（记忆→理解→应用→分析→评价→创造）
4. **题型复杂度递增**：难度不仅体现在知识点深度，也体现在题型综合性

## 难度等级定义（通用框架）

| 等级 | 名称 | 认知层次 | 特征 | 适用场景 |
|------|------|----------|------|----------|
| 1 | 基础识记 | 记忆 | 直接考查概念、定义、公式，无变形 | 课堂练习 |
| 2 | 简单理解 | 理解 | 简单应用基本概念，一步计算 | 随堂小测 |
| 3 | 基础应用 | 应用 | 套用公式或方法，2-3步计算 | 单元测试基础题 |
| 4 | 熟练应用 | 应用 | 需要选择合适方法，3-4步计算 | 单元测试中档题 |
| 5 | 综合应用 | 分析 | 多知识点结合，需要分析题意 | 期中期末中档题 |
| 6 | 灵活变通 | 分析 | 题目有变形，需要转化思维 | 期末压轴题前置 |
| 7 | 深度分析 | 分析/评价 | 多步骤推理，需要构建解题思路 | 中考/高考中档题 |
| 8 | 综合创新 | 评价/创造 | 跨章节综合，需要创造性思维 | 中考/高考压轴题 |
| 9 | 竞赛入门 | 创造 | 需要巧妙方法或非常规思路 | 竞赛初赛 |
| 10 | 竞赛难题 | 创造 | 需要深厚功底和创新能力 | 竞赛决赛 |

## 年级×科目难度矩阵示例

### 小学数学难度细化

**1级（基础识记）**
- 题型：直接计算、认识图形、读数写数
- 示例：3 + 5 = ?
- 知识点：10以内加减法、认识钟表
- 时间：30秒/题

**5级（综合应用）**
- 题型：应用题（两步计算）、图形面积计算
- 示例：小明有15个苹果，吃了3个，又买了8个，现在有多少个？
- 知识点：多步计算、单位换算
- 时间：3-5分钟/题

**10级（竞赛难题）**
- 题型：奥数题、数论问题、组合问题
- 示例：用1-9九个数字填入九宫格，使每行每列每对角线和相等
- 知识点：数学思维、逻辑推理
- 时间：10-15分钟/题

### 高中数学难度细化

**1级（基础识记）**
- 题型：概念判断、公式默写
- 示例：sin²x + cos²x = ?
- 知识点：三角函数基本公式
- 时间：30秒/题

**5级（综合应用）**
- 题型：函数性质综合、三角恒等变换
- 示例：已知f(x)=sin(2x+π/3)，求f(x)的单调递增区间
- 知识点：三角函数图像与性质
- 时间：5-8分钟/题

**10级（竞赛难题）**
- 题型：导数压轴题、解析几何综合
- 示例：已知函数f(x)=e^x-ax²，讨论f(x)的零点个数
- 知识点：导数、函数零点、分类讨论
- 时间：15-20分钟/题

## 难度控制提示词模板

```javascript
// 根据难度等级生成对应的提示词片段
function getDifficultyPrompt(difficulty, grade, subject) {
  const level = DIFFICULTY_MATRIX[grade][subject][difficulty];

  return `
【难度要求】
- 难度等级：${difficulty}/10（${level.name}）
- 认知层次：${level.cognitive}
- 题目特征：${level.features}
- 解题步骤：${level.steps}
- 预计用时：${level.time}
- 知识点深度：${level.depth}
- 题型复杂度：${level.complexity}

【出题约束】
${level.constraints.map(c => `- ${c}`).join('\n')}

【参考示例】
${level.example}
`;
}
```

## 难度梯度控制策略

### 选择题难度控制
- 1-3级：选项差异明显，干扰项容易排除
- 4-6级：选项相似度提高，需要仔细计算
- 7-10级：选项设置陷阱，需要深度理解

### 解答题难度控制
- 1-3级：单一知识点，步骤明确
- 4-6级：2-3个知识点结合，需要选择方法
- 7-10级：多知识点综合，需要构建解题框架

### 填空题难度控制
- 1-3级：直接填写答案，无需过程
- 4-6级：需要中间计算，答案唯一
- 7-10级：需要分类讨论，可能多解

## 难度校验机制

生成题目后，AI需要自我评估：
1. 该题目是否符合目标难度等级？
2. 解题步骤数是否匹配？
3. 知识点深度是否合适？
4. 预计用时是否合理？

如果不符合，重新生成。
